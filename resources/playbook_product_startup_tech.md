# 骨架一：产品深度拆解

适用场景：对一款科技产品进行极客视角的深度体验与设计逻辑拆解。

## 结构模板

```
【开场：体验的第一印象】
用「我第一次用它时」的第一人称场景开场：
- 什么地方让你「意外」了？
- 什么设计决策是你没想到的？

【这款产品的「极客亮点」在哪里】
聚焦2-3个技术/设计上的不寻常之处：
- 「大多数产品选择[A]，而它选择了[B]，原因是[逻辑]」
- 工程师或设计师视角的深层分析

【和竞品的本质区别】
不是功能对比表，而是「设计哲学的差异」：
- 它认为用户最需要的是什么？
- 竞品认为用户最需要的是什么？
- 这两种判断背后的用户洞察差异

【这款产品的局限在哪里】
诚实的极客评估：
- 哪里做得还不够好？
- 功能/性能/易用性的真实短板

【期待它的下一版本做什么】
以极客用户的期待结尾，而不是广告式收尾
```

# 骨架二：早期创业公司发现

适用场景：报道一家还不太出名但值得关注的早期创新公司。

## 结构模板

```
【发现感：为什么今天要写这家公司】
不是「X公司宣布做Y」，而是：
「我第一次听到[公司名]是从[场景/人物]，当时觉得[反应]。」

【他们在做一件什么事】
用最简洁的语言定义：[用户] + [核心痛点] + [解决方案]

【为什么是这个团队】
「找到这些人，是理解这家公司的关键」：
- 团队背景与问题的匹配度
- 什么经历让他们特别适合做这个？

【早期验证信号】
哪些早期数据/用户反应支持这个方向：
- 不一定是大数字，但要是「真实的信号」

【还需要证明什么】
诚实的风险评估：
- 这家公司要走到下一阶段，还需要证明哪些事？
```

# 骨架三：技术趋势深度

适用场景：对一个正在成熟的技术趋势进行极客视角的深度分析。

## 结构模板

```
【这项技术到了哪个发展阶段】
用「Gartner炒作周期」框架定位：
- 现在是：萌芽期/过热期/幻灭期/复苏期/成熟期？

【技术的真实现状（剥离炒作）】
区分「能做到的」和「说能做到的」：
- 实验室指标 vs 工程落地现实
- 最佳使用场景 vs 媒体过度承诺的场景

【谁在做，做到了什么程度】
梳理技术应用的真实进展：
- 学术/研究机构的最新突破
- 工业界的落地应用（可同时列国内外）

【还差什么才能成为主流】
技术成熟度的gap分析：
- 成本/算力/数据/监管/用户教育，哪个是主要瓶颈？

【极客展望：值得持续关注的信号】
给极客读者带走「追踪清单」
```
