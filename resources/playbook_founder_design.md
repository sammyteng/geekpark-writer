# 骨架四：创始人叙事

适用场景：以极客公园的视角写一位科技创业者，聚焦其产品哲学与创新思维。

## 结构模板

```
【开场：一个产品决策，折射这个人的思维方式】
不从简历不从背景，从一个具体的「产品判断」开场：
「[创始人]曾做了一个[外界不理解的]决定。
 现在来看，这件事道出了他对[行业/用户/技术]的理解。」

【他/她是如何「看世界」的】
揭示创始人的底层认知模型：
- 他们觉得行业最大的错误假设是什么？
- 他们「看到了」其他人没看到的什么？

【产品哲学：从产品的设计看创始人的信念】
通过产品本身来呈现创始人理念：
- 「[功能X]的设计，体现了他[信念Y]」
- 「他们刻意不做[某类竞品常见功能]，因为[背后判断]」

【和极客社区的关系】
极客公园特色：这个创始人和极客用户的互动方式：
- 是否参与了技术社区讨论？
- 是否开源了部分技术？
- 极客用户对他们的评价是什么？

【这个人眼中的「下一件大事」】
结尾聚焦他/她现在最感兴趣的问题
不是采访稿式引用，而是从行为推断：「他们正在做的[X]，暗示了他们认为[Y]」
```

# 骨架五：产品设计决策解读

适用场景：以工程师/设计师视角解读一款产品中特定的设计决策。

## 结构模板

```
【这个设计决策是什么】
用一句话定义本文聚焦的具体决策：
「这篇文章只说一件事：[产品]为什么选择了[具体设计选择]。」

【大多数人怎么做，为什么】
建立对比基准：
- 同类产品/竞品的通常做法是什么？
- 这个「通常做法」背后的逻辑是什么？

【他们不同在哪里，为什么】
核心分析层：
- [产品]选择了[不同的做法]
- 背后的第一原理逻辑是[X]
- 这个选择有什么代价？有什么收益？

【用户真实反应】
引用真实的用户反馈验证决策效果：
- App Store 评论 / 社区讨论 / 用户访谈引用
- 数据层面的验证（如果有公开数据）

【这个决策对行业的启示】
从单一决策提炼更广泛的产品设计洞察：
「[这个决策]说明：在[某类场景]下，[设计原则X]比[设计原则Y]更重要」
```
