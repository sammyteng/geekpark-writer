# 🤖 极客公园 Writer Skill

> 让 AI 以极客公园「极客发现感+科技创新叙事」风格写科技深度报道的可复用 Skill。

## 简介

本 Skill 基于对极客公园数百篇科技创新报道的结构化解析，将极客公园「以极客视角发现改变力、深挖产品与创业故事」的写作方式标准化为可复用 AI 写作指令。

## 文件结构

```
geekpark-writer/
├── SKILL.md                        # 主技能文件（核心写作哲学 + 流程）
└── resources/
    ├── playbook_product.md         # 骨架一：产品深度拆解
    ├── playbook_startup.md         # 骨架二：早期创业公司发现
    ├── playbook_tech.md            # 骨架三：技术趋势深度
    ├── playbook_founder.md         # 骨架四：创始人叙事
    └── playbook_design.md          # 骨架五：产品设计决策解读
```

## 极客公园核心写作风格

| 维度 | 特征 |
|------|------|
| 开场 | 「发现感」场景，记者第一视角 |
| 核心 | 产品/技术设计决策的深层逻辑 |
| 创业叙事 | 为什么是这个团队做这件事 |
| 专业度 | 技术描述准确，工程师觉得对、外行也能懂 |
| 禁忌 | 功能列表式介绍、通稿化视角、只关注已成功公司 |

## 核心写作哲学

- **发现感优先**：这家公司/产品「为什么值得今天被报道」
- **产品设计思维**：拆解「为什么这样设计」而不只是「做了什么」
- **创业原点故事**：「为什么是这些人做这件事」
- **极客精神**：尊重技术，用第一原理理解创新

## 如何使用

```
使用 geekpark-writer，帮我用极客公园风格写一篇关于 [产品/公司/技术] 的报道
```

## 适合场景

✅ 新产品/新技术的深度体验报道
✅ 创新型早期公司的发现式报道
✅ 产品设计决策的极客视角解读
✅ 创始人深度人物叙事
